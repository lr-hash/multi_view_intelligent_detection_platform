# Specification - 报警历史与多维数据查询

## 1. 概述
本轨道旨在完善系统的辅助功能，为用户提供对系统运行过程中的异常报警记录进行追溯的能力，并提供一个通用的多维数据查询工具，方便技术人员调取历史监测数据进行分析。

## 2. 需求分析
### 2.1 报警记录持久化
- **数据库模型**：创建 `AlarmRecord` 模型，记录报警时间、类型（矿压/微震/变形）、等级（红/黄）、具体数值及处理状态。
- **自动触发**：在数据集成处理过程中，若数值超过阈值，自动写入报警表。

### 2.2 多维数据查询
- **综合查询接口**：支持按时间段、工作面/钻孔、数值范围进行数据检索。
- **覆盖数据项**：矿压原始数据、微震事件、巷道变形记录、压裂施工详情。

### 2.3 前端辅助界面
- **报警历史页面 (`AlarmHistoryView.vue`)**：支持筛选、确认（解除）报警操作。
- **综合查询页面 (`DataQueryView.vue`)**：提供高级过滤器，支持查询结果的表格导出。

## 3. 验收标准
- [ ] 后端成功创建 `AlarmRecord` 模型并应用迁移。
- [ ] 数据摄入时能根据预设阈值正确触发并保存报警记录。
- [ ] 查询接口能正确响应复杂条件过滤（如：查询能量 > 1000J 且在过去 3 天内的微震事件）。
- [ ] 前端页面能展示查询结果，布局清晰易用。
