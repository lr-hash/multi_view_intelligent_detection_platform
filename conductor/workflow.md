# Development Workflow

## 1. 任务开发流程
每个任务（Task）都应遵循以下循环：
1. **理解 (Understand)**：明确任务目标及验收标准。
2. **计划 (Plan)**：在 `plan.md` 中细化步骤。
3. **实现 (Implement)**：编写代码。对于功能开发，遵循 TDD（测试驱动开发）原则：先写测试，再写实现。
4. **验证 (Verify)**：运行测试并进行代码风格检查（Linting）。
5. **提交 (Commit)**：任务完成后，提交代码并更新 `plan.md` 状态。

## 2. 质量标准
- **测试覆盖率**：核心逻辑的单元测试覆盖率应达到 80% 以上。
- **代码规范**：所有代码必须通过对应语言的 Lint 检查。
- **手动验证**：UI 更改需经过手动界面检查。

## 3. 提交规范
- 每个任务完成后执行 Git Commit。
- 使用 `git notes` 记录任务的详细摘要和完成情况。
- 阶段性（Phase）完成后，进行一次总结性提交。

## 4. 阶段验收协议
在每个阶段（Phase）结束时，必须执行“Conductor - 用户手动验证”任务，确保该阶段的所有功能符合 `spec.md` 的描述。
