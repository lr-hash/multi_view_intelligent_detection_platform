# Specification - 系统管理、权限控制与配置界面

## 1. 概述
本轨道旨在完善平台的安全性与管理能力。通过引入 JWT 认证机制确保数据安全，并提供一个管理界面，允许管理员动态调整系统运行参数（如数据接口地址、频率）和安全报警阈值。

## 2. 需求分析
### 2.1 身份验证与权限 (Auth & RBAC)
- **JWT 认证**：后端实现 `/auth/login` 接口，验证成功后返回令牌。前端请求需在 Header 中携带 Token。
- **角色区分**：定义 Admin（管理员）和 User（普通用户）角色。
- **路由保护**：敏感操作（如修改配置）仅限 Admin 权限。

### 2.2 系统配置管理 (System Config)
- **接口参数配置**：支持对外部系统（KJ653, SOS, 压裂设备）的 IP、端口、采集频率进行在线修改。
- **阈值配置**：支持对矿压、微震、变形报警的红/黄阈值进行在线动态调整。
- **持久化**：配置信息存储在数据库中，并在服务重启后自动加载。

### 2.3 管理界面
- **登录页面**：实现基础的登录表单。
- **配置看板**：展示当前系统参数，并提供修改入口。

## 3. 验收标准
- [ ] 未携带有效 Token 的请求无法访问受保护的接口。
- [ ] 管理员用户能够成功修改报警阈值，且后端逻辑立即采用新阈值。
- [ ] 接口配置修改后，后端集成服务能获取到最新的连接参数（模拟体现）。
- [ ] 前端实现登录拦截与权限路由控制。
